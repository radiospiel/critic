We welcome contributions! This document explains how you can get involved.

## Code Contributions

Code contributions are welcome via pull requests. However, to keep the project's licensing clean and maintainable, **all code contributions require a signed [Contributor License Agreement (CLA)](CLA.md)**.

### Why a CLA?

A CLA clarifies the intellectual property rights around contributions. It ensures that:

- You confirm you have the right to submit the contribution.
- The project maintainer can continue to manage licensing consistently.
- The project can be defended against IP disputes.

### How it works

1. Before your first contribution is merged, you will be asked to sign the CLA.
2. The CLA only needs to be signed once.
3. Details on the CLA and how to sign it will be provided when you open your first pull request, or you can reach out to the maintainer at https://github.com/radiospiel.

## Non-Code Contributions

There are many valuable ways to contribute that don't involve writing code:

- **Bug reports** — Found something broken? Open an issue with steps to reproduce.
- **Feature requests** — Have an idea for an improvement? We'd love to hear it.
- **Documentation** — Help improve docs, fix typos, or write guides.
- **Testing** — Try things out and report what works and what doesn't.
- **Discussion** — Participate in issues and help other users.

Non-code contributions do not require a signed CLA.

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make your changes.
4. Open a pull request with a clear description of what you've done and why.

For questions, reach out to the maintainer via https://github.com/radiospiel.
