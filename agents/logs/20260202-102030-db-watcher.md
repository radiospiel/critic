# Add Database Watcher for Live Reload

**Started:** 2026-02-02 10:20
**Ended:** 2026-02-02 10:22
**Complexity:** Medium
**Outcome:** Complete

## Task
Watch the .critic directory for database changes. When we detect a change:
1. Fetch the latest `_db_mtime` for the messages table
2. When that has changed, trigger a reload of the file list and diff view

## Strategy
Feature - New functionality

## Implementation Plan
1. Add `GetMessagesMtime()` method to messagedb.DB
2. Create `DBWatcher` in api/server that:
   - Uses FileWatcher to watch critic.db
   - Tracks last known mtime
   - On file change, queries mtime and compares
   - Sends notifications when mtime has changed
3. Add watcher to server startup and handle changes
4. Broadcast reload on database changes
