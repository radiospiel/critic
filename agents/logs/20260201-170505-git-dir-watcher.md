## Objective
When starting the server, also start a file watcher that watches the `.git` directory for changes. When changes are detected:
1. Update the timestamp returned from `GetLastChange`
2. Send a "reload" signal to the webui via websocket

## Design Notes
- The existing `Watcher` explicitly skips `.git` directory (designed for working dir changes)
- Created a separate, simpler `GitWatcher` specifically for `.git` directory
- GitWatcher watches recursively including subdirectories like `refs/`, etc.
- Uses 100ms debounce to handle rapid git operations
